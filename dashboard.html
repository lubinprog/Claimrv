Comment ça fonctionne
Chaque application de formulaire envoie ses soumissions à une URL webhook unique. Tu colles cette URL dans ton app, et chaque formulaire soumis par un technicien crée automatiquement un claim dans LUKANE RV — sans aucune manipulation manuelle.

FastField Forms
Plan Business ou Enterprise requis (webhooks HTTP/HTTPS)
1
Connecte-toi au portail FastField
Va sur app.fastfieldforms.com et ouvre ton formulaire "BT tâche en garantie" (ou ton formulaire de claim).
2
Ouvre le Form Builder
Clique sur Edit Form puis sur le bouton bleu Delivery en haut à droite.
3
Ajouter une action HTTP/HTTPS
Dans la colonne Form Submit Actions, glisse le bloc HTTP/HTTPS dans la zone d'actions.
4
Colle l'URL webhook
Dans le champ URL, colle ton URL LUKANE RV. Sélectionne le format JSON. Méthode : POST.
https://lukanerv.vercel.app/api/webhook // cliquer pour copier
5
Nomme tes champs correctement
Assure-toi que tes questions dans FastField contiennent ces mots-clés dans leur titre (insensible à la casse) :

Webhook JSON personnalisé
Pour tout système qui envoie du JSON via HTTP POST
1
Configure ton système
Ton système doit faire un HTTP POST vers l'URL LUKANE RV avec un Content-Type: application/json.
2
Format JSON attendu
Envoie un objet JSON avec les champs ci-contre. Tous les champs sont optionnels sauf mfr et desc.
3
Authentification (optionnel)
Ajoute le header X-API-Key: VOTRE_CLE pour sécuriser les requêtes.
4
Test avec cURL
Copie cette commande pour tester immédiatement :
curl -X POST https://lukanerv.vercel.app/api/webhook \ -H "Content-Type: application/json" \ -d '{"mfr":"Grand Design","unit":"Reflection 303RLS","vin":"4X4GD2025001","year":"2025","category":"Slide room","desc":"slide sort pas","tech":"Marc","bt":"BT-001"}'
Format JSON attendu
{
  "mfr": "Grand Design",
  "unit": "Reflection 303RLS",
  "vin": "4X4GD2025001",
  "year": "2025",
  "category": "Slide room",
  "desc": "slide sort pas...",
  "tech": "Marc Tremblay",
  "bt": "BT-2025-001"
}
